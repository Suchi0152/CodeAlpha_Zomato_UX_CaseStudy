# 🍽️ Zomato – UX Case Study

A UX research and analysis case study on the **Zomato** food delivery app (Android & iOS), conducted as part of the CodeAlpha UI/UX Design Internship.

## 📋 Project Details

- **Submitted by:** Kanigelpula Suchitra
- **Organization:** CodeAlpha Internship Program
- **Task:** Task 3 — UX Case Study
- **Platform Analyzed:** Zomato (Android version 17.x)
- **Date:** June 2025

## 🎯 Objective

- Understand Zomato's current user journey from app open to order delivery
- Identify usability issues and UX friction points
- Suggest 3 evidence-based improvements with mockup descriptions

## 🗺️ User Journey Analysis

The case study maps Zomato's user journey across 6 key stages:

1. **App Open & Location Detection**
2. **Browse & Discover**
3. **Restaurant Selection**
4. **Cart & Customization**
5. **Checkout & Payment**
6. **Order Tracking & Delivery**

## ✅ Key Strengths Identified

- Real-time, map-based order tracking
- Powerful search & filtering (cuisine, rating, price, dietary preference)
- Personalization via "Reorder" and AI-driven recommendations
- Well-integrated Zomato Pro (Gold) loyalty program
- Trust-building rating & review system with photo reviews

## ⚠️ Key Pain Points Identified

- Cluttered home screen with excessive ads/banners
- Checkout flow friction from repeated upsell prompts
- Cart silently clears when switching restaurants
- Inconsistent loading states causing layout shifts
- Notification overload with buried granular controls

## 💡 Proposed UX Improvements

| # | Improvement | Problem Solved | Expected Impact |
|---|------------|-----------------|------------------|
| 01 | Simplified & Personalized Home Screen ("Focus Mode") | Ad/banner clutter, decision fatigue | Faster decisions, higher conversion |
| 02 | Streamlined 3-Step Checkout | Multi-screen checkout with upsell interruptions | ~40% faster checkout, less abandonment |
| 03 | Smart Cart Persistence & Conflict Resolution | Cart silently clears on restaurant switch | Less abandonment, higher satisfaction |

## 📊 Conclusion

Zomato is a mature, well-designed product with strong core functionality, but has accumulated UX debt — especially in home screen information architecture and checkout flow. The proposed improvements target high user impact with relatively low implementation complexity, aiming to improve checkout conversion, session duration, CSAT/NPS scores, and cart abandonment rates.
